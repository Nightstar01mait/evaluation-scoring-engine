# Project Overview

The **Evaluation Scoring Engine** is a Python-based system designed to evaluate interview responses using a hybrid scoring approach.

The system combines:

- Rule-based evaluation
- Large Language Model (LLM) evaluation
- Machine Learning scoring
- Performance analytics

The goal of this system is to simulate a structured interview evaluation process and generate fair scoring with actionable feedback.

---

# Key Features

## 1. Multi-Dimensional Scoring

Candidate responses are evaluated across four dimensions:

- Technical Accuracy
- Concept Clarity
- Keyword Coverage
- Communication Quality

Each dimension contributes to the final evaluation score.

---

# 2. Experience-Based Difficulty Scaling

The system adjusts evaluation difficulty based on the candidate’s experience level.

| Experience Level | Multiplier |
|------------------|-----------|
| Fresher | 1.0 |
| Intermediate | 1.1 |
| Advanced | 1.2 |

Final Score Formula

```
Final Score = Base Score × Experience Multiplier
(Max score capped at 10)
```

---

# 3. Experience-Based Passing Criteria

The system determines whether a candidate passes or fails based on **overall performance and experience level**.

| Experience Level | Passing Score |
|------------------|--------------|
| Fresher | 5.0 |
| Intermediate | 6.0 |
| Advanced | 7.0 |

This ensures fair evaluation across different skill levels.

---

# 4. Candidate Suggestions System

The system provides **automatically generated suggestions** to help candidates improve their interview responses.

Suggestions are generated based on weaknesses detected in different evaluation dimensions.

The goal of this feature is to provide **actionable feedback** rather than only a score.

---

## Suggestion Generation Logic

The system analyzes the candidate response using multiple scoring dimensions:

- Technical Accuracy
- Concept Clarity
- Keyword Coverage
- Communication Quality

If any of these scores fall below a defined threshold, the system generates targeted suggestions.

---

## Example Suggestions

Typical suggestions generated by the system include:

- Focus on explaining concepts more clearly.
- Include more relevant technical keywords.
- Structure the answer more logically.
- Provide examples to support explanations.
- Improve communication and clarity of expression.

---

## Example Output

```
Suggestions:

- Focus on explaining concepts more clearly.
- Include more relevant technical keywords.
- Work on communication and structuring your answer better.
```

These suggestions help candidates understand **where they need improvement**.

---

## Implementation

Suggestions are generated during the evaluation process and stored in the evaluation output object.

Example structure:

```
suggestions = [
    "Focus on explaining concepts more clearly",
    "Include more relevant technical keywords",
    "Improve communication clarity"
]
```

These suggestions are displayed after the evaluation result and also included in the performance report.

---

## Benefits

- Provides constructive feedback
- Helps candidates improve answer quality
- Simulates real interview feedback systems
- Improves learning outcomes
---

# 5. Hybrid LLM + ML Evaluation

The evaluation combines:

- LLM based semantic understanding
- ML based scoring prediction
- Weighted scoring logic

This hybrid approach improves reliability and fairness.

---

# 6. Dummy Machine Learning Model

A dummy ML model is implemented to simulate automated answer scoring.

The model is trained on an essay scoring dataset using:

- **TF-IDF Vectorization**
- **Regression-based scoring model**

The model predicts answer quality based on textual features.

# Installation

Clone the repository:

```bash
git clone https://github.com/Nightstar01mait/evaluation-scoring-engine.git
cd evaluation-scoring-engine
```

Create virtual environment:

```bash
python -m venv .venv
```

Activate virtual environment:

Windows

```bash
.venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

# Dataset

The dataset used for training is not included in this repository due to GitHub file size limitations.

Recommended dataset:

https://www.kaggle.com/datasets/spsayakpaul/automated-essay-scoring-dataset

After downloading the dataset, place it in:

```
data/essay_dataset.csv
```

---

# Dataset Format

The training script expects a CSV file with the following format:

```
text,score
"This essay explains machine learning concepts...",8
"Short explanation about AI...",5
"Very poor answer...",2
```

| Column | Description |
|------|-------------|
| text | Candidate answer text |
| score | Ground truth score (0–10) |
